## 严格遵循的任务

1. 修改某段代码是极其慎重的行为，必须得到使用者的 “显式指定或者许可” 才能执行。
2. 比如某个场景下，你有修改代码的打算，你必须提出，“是否允许 LLM 修改代码”这样的指令。然后得到使用者的明确认可，才能修改。
3. 如何定义“明确认可” --> 必须是严格的 code 这样的四个字母。我们可以按照一些场景来举例说明：

- 是否允许 copilot 修改代码 --> Wrong 因为你的请求只能是 “是否允许 LLM 修改代码” 这样一个严格的字符串，需要严格匹配。
- 是否允许 LLM 修改代码 --> 我的应答：yes --> 拒绝执行修改代码的指令，因为我的应答必须是 code 这样严格的 4 个字母，否则一律拒绝执行。
- 是否允许 LLM 修改代码 --> 我的应答：允许 --> 拒绝执行修改代码的指令，因为我的应答必须是 code 这样严格的 4 个字母，否则一律拒绝执行。
- 是否允许 LLM 修改代码 --> 我的应答：code --> 允许执行。 4.除此以外，严格禁止修改哪怕一行代码。

---

5.如果我显式指定你：“去修改 xxx 文件的代码”，那么你应该向我提问：是否允许 LLM 修改代码，然后我回答 code 你才可以执行，否则一律严格禁止修改代码。
